# PackSmart — бирки для пильного центра (полное описание проекта)

Документ-«паспорт» системы: что это, как устроено, где данные, как менять, как чинить.
Секретных значений (токенов) здесь нет — только где они лежат.

---

## 1. Что это и зачем

Система автоматически делает **бирки (этикетки)** для деталей мебели, которые режутся на
**пильном центре**, и кладёт их в **Google Drive**, откуда станок их печатает.

**Идеальный поток (то, ради чего всё сделано):**
оператор добавляет задачи в таблицу на новый день → BAZIS (раскрой) кладёт XML в папку Drive →
**бирки в Drive появляются и обновляются сами** (место/дата/кол-во/материал/кромка из таблиц).
Поменял «Место» в таблице → через несколько минут бирки в Drive обновились. Без кликов.

## 3. Откуда какие данные на бирке

Бирка (макет по эскизу): QR • Наименование • Дата • Кол-во в партии • Материал • **Место** • Длина×Ширина • CNC.

| Поле | Источник |
|---|---|
| **Наименование** | имя детали (РАСПИЛ «Наименование» / папка XML); `/` → `-` |
| **Дата** | РАСПИЛ, колонка A «Дата» (формат дополняется до dd.mm.yyyy) |
| **Кол-во в партии** (число на бирке) | РАСПИЛ **Д1 (колонка E)** — число `Nшт` из строки `Имя-Nшт \| ячейка` |
| **число файлов бирок** (label1..labelN) | из **XML** (сколько деталей в раскрое) |
| **Материал** | Details List «Материал» (короткое имя: `ЛДСП Белый Молет` → `ЛДСП «Белый»`) |
| **Место** (ячейка) | Details List «Место» — главное динамическое поле |
| **Длина × Ширина** | Details List «Длина», «Ширина» |
| **Кромка** (чёрточки под размерами) | Details List **ТолК / ДК / ШК** (см. ниже) |
| **CNC** (иконка) | Details List «Присадка» (заполнено → рисуем `assets/cnc_prisadka.png`) |
| **QR** | компактный текст со всеми полями (имя, дата, кол-во, материал, место, размер, кромка, CNC) |

**Кромка (для кромкооблицовочного станка):**
- **ДК** → линии под **Длиной**, **ШК** → под **Шириной**: `0`/пусто = нет, `1` = одна, `2` = две параллельные.
- **ТолК** → толщина линий: `0.8` = жирная, `0.4` = тонкая.

**QR — компактный формат** (читается телефоном), пример:
```
F720x400 белый
19.06.2026 · 84шт
ЛДСП «Белый»
Место: B3
716x396
Кромка: Д2/Ш0, 0.8
CNC: да
```

**Формат файла бирки `.emf`:** монохромный BMP 1419×884 (1-bit), записанный **дважды подряд**
(итого 318 364 байта) — так его понимает станок. Расширение `.emf`, но внутри BMP.

## 4. XML раскроя (BAZIS) — как из него берём бирки

Пример детали в XML: `code="labels\ANT-Bok L-R-ЛДСП «Белый»\label1"`.
- **Имя папки** = между `labels\` и `\labelN` (= `<Деталь>-<Материал>`).
- **Число бирок** = сколько разных `labelN` в XML (напр. label1..label27 → 27).
- Имя детали = папка минус `-<материал>`; ищется в Details List по имени (с заменой `/`↔`-`).

## 5. Две части системы (важно не путать)

### 5.1 Сайт (папка `docs/`, GitHub Pages) — «лицо» для человека
- Открыл ссылку → видит задание на сегодня (если нет — последнее), бирки сеткой.
- Кнопки: **Печать всех**, **Скачать всё (PDF)**, **Скачать всё (EMF, ZIP)**, поиск по названию/месту/материалу.
- Блок **«Бирки по XML»**: выбрал XML → превью + **Скачать папку (ZIP)** с label1..labelN.
- Данные читает **вживую в браузере** (gviz JSONP). Сам **ничего не хранит** и **в Drive не пишет**.
- Печать/PDF — раскладка на лист **A4** сеткой; размер бирки `labelWidthMm` в `docs/config.js` (по умолч. 90 мм).

### 5.2 Синхронизация (`sync_drive.py` + GitHub Actions) — «мотор», автоматика
- Запускается **по расписанию каждые 5 минут** (`.github/workflows/sync.yml`, cron `*/5 * * * *`).
- Читает **свежие XML** (за последние `SYNC_DAYS` дней, по умолч. 7; сортировка — новые первыми) из папки XML.
- Берёт данные из таблиц, рендерит бирки (тот же движок, что и сайт).
- Кладёт N бирок в подпапку внутри папки `labels` в Drive.
- **Обновляет только изменённое** (сравнение по md5; если совпало — пропускает).
- Обновляет файлы **на месте** (`files.update`), а НЕ удаляет — потому что чужие файлы (папка в «Доступные мне») удалять нельзя.
- Устойчив к ошибкам (одна деталь не роняет весь прогон) и к лимитам Google (`num_retries=5`).

## 6. Ключи и где они лежат

Все ключи — в файле **`.env`** (в корне, **в .gitignore**, в репозиторий НЕ попадает):
```
GOOGLE_CLIENT_ID=...
GOOGLE_CLIENT_SECRET=...
GOOGLE_REFRESH_TOKEN=...
RASPIL_SHEET_ID=10nsX_nmzfPB7IyHM7oSqlj8wrdBn-zfoaKiMDZSccKI
```
- Шаблон без значений — `.env.example` (он в репозитории).
- Код подхватывает `.env` при старте (`generate_labels.py` → `_load_dotenv`). Реальные переменные
  окружения (например, **GitHub Secrets**) имеют приоритет над `.env`.
- На **GitHub** те же ключи лежат в **Secrets** (Settings → Secrets → Actions): `GOOGLE_CLIENT_ID`,
  `GOOGLE_CLIENT_SECRET`, `GOOGLE_REFRESH_TOKEN`. Прочитать их в GitHub нельзя (только перезаписать).
- `drive-auto/client_secret.json` — OAuth-клиент (Desktop), **только локально**, в .gitignore.
- Получить/обновить refresh-токен: `python3 drive-auto/get_token.py` (откроет браузер). См. `drive-auto/SYNC_SETUP.md`.

## 8. Как делать частые операции

- **Поменять место/материал/размер/кромку детали** → правишь **Details List** (вкладка Main).
- **Поменять кол-во/дату/добавить задачи** → правишь **РАСПИЛ**.
  → Бирки в Drive обновятся при следующем прогоне (~5 минут). Срочно: GitHub → **Actions** →
  «Синхронизация бирок в Drive» → **Run workflow**.
- **Полный прогон по всей истории** (все XML): Actions → Run workflow → поле **days = all** (идёт долго).
- **Изменить интервал**: `.github/workflows/sync.yml`, строка `cron`. На GitHub минимум — 5 минут.
- **Деталь не делается** («нет в базе») → её надо добавить в **Details List**.
- **Запустить вручную локально**: `python3 sync_drive.py` (нужен `.env` с ключами + `pip install pillow segno google-api-python-client google-auth`).
- **Сделать бирки по одному XML локально**: `python3 generate_labels.py --xml файл.xml`.
- **Бирки за дату локально**: `python3 generate_labels.py --job 08.12.2025`.

## 9. Деплой «мотора» на свой сервер (вместо/вместе с GitHub)

Зачем: интервал **1 минута** (на GitHub минимум 5) и без задержек GitHub. Сервер должен быть всегда включён.

1. `git clone https://github.com/Kazhymukhann/raspil-labels.git`
2. **Скопировать `.env`** на сервер (в корень проекта) — он в git не входит, переносится вручную.
3. Установить: `pip3 install pillow segno google-api-python-client google-auth` и шрифты Arial
   (`sudo apt-get install -y ttf-mscorefonts-installer`).
4. Добавить в `cron` (каждую минуту):
   `* * * * * cd /путь/raspil-labels && python3 sync_drive.py >> sync.log 2>&1`
5. (рекомендуется) ротация `sync.log`, чтобы не разрастался.
6. Если переносишь полностью — можно отключить расписание на GitHub (убрать `schedule` в `sync.yml`).

## 10. Известные нюансы / подводные камни

- **Превью в Drive «не меняется».** При обновлении файла «на месте» Google показывает **старую
  картинку-превью** (кэш), хотя содержимое уже новое. Станок читает содержимое — у него всё свежее.
  Проверять — **скачать** файл и открыть, а не смотреть плитку.
- **Чужие файлы не удаляются.** Папка `labels` в «Доступные мне» (владелец — другой аккаунт),
  поэтому бирки **обновляются**, а не пересоздаются; лишние старые файлы могут остаться.
- **GitHub cron задерживается** под нагрузкой (5 мин могут стать 7–15). Свой сервер — точнее.
- **Apps Script-путь заброшен**: у аккаунта нет «анонимного» доступа к web-app, а кросс-доменные
  куки Google не шлются с сайта — поэтому заливка через Apps Script не работала. Используем
  GitHub Actions + OAuth.
- **Приватность**: ID таблиц публичны (в коде сайта), таблицы открыты по ссылке — теоретически их
  может прочитать любой, у кого есть ID (включая вкладки с ценами/ЗП). Если надо закрыть —
  публиковать только нужные вкладки как CSV и читать их, либо закрыть доступ и пускать только через токен.
- **Детали без места в Details List** пропускаются (на момент написания не хватало:
  F360x500 Белый; F450x600/400 Белый; Plintus 800/600/500/400 Серый; F240x600 Серый; F240х400 Серый).

## 11. Быстрая диагностика

- Прогон упал/не залил → GitHub **Actions** → открыть последний запуск → лог шага «Синхронизация»
  (там по каждой детали: `✓ ... кол-во на бирке ...` / `? нет в базе` / `! ошибка`).
- «Бирки старые на вид» → это кэш превью; скачать файл и проверить содержимое.
- «Ничего не синхронизируется» → проверить, что в GitHub Secrets заданы 3 ключа GOOGLE_*,
  и что у аккаунта есть доступ Редактор к обеим папкам Drive.
