## Что происходит на линии (как это работает)

1. **Пильный центр** (участок №1) получает задание в виде **XML** — это программа реза
   (порядок, как пилить лист). Внутри XML прописаны **ссылки на файлы бирок** в подпапке
   `labels/` (относительный путь). Поэтому:
   - если загрузить только XML — станок «не видит бирки» (путь есть, файлов нет);
   - как только рядом лежит папка `labels/` с `label1.emf, label2.emf, …` — станок
     находит их по пути и печатает на принтере, когда деталь вышла из станка.

2. **Файл бирки `labelN.emf`** — это НЕ «живой» шаблон, а **готовая картинка**.
   По факту это монохромный **BMP `1419×884`, 1‑bit**, записанный ДВАЖДЫ подряд
   (две одинаковые копии) и с расширением `.emf`. Текст «№ячейки», название, размеры —
   всё «впечатано» в пиксели. Чтобы поменять ячейку — картинку надо **перерисовать**.

3. **Ячейка («место»)** — это адрес A1/B3/E3/H8/X на складе/стеллаже. Хранится в базе
   (Google‑таблица, вкладка **BASE**, колонка **«Место»**). Поменял там — нужно
   перегенерировать бирку, и новая ячейка появится на печати.

## База (источник данных)

Google Sheet, вкладка **BASE** — по одной строке на деталь:

| Детали | Материал | **Место** | Присадка | Тип | Длина | Ширина | Толщина | … |
|--------|----------|-----------|----------|-----|-------|--------|---------|---|
| ANT-Bok L/R | ЛДСП Белый Молет | **E3** | Присадка | Кухня | 280 | 560 | 16 | … |

На бирку идёт (финальный макет, по эскизу):
QR • Наименование по системе • Дата • Кол-во в партии • Материал •
**Место (= колонка «Место», динамика)** • Длина×Ширина • иконка CNC (если есть «Присадка»).

Источники полей:
- из **BASE**: Материал, **Место**, Длина, Ширина, признак CNC (Присадка);
- из **задания** (вход/РАСПИЛ/XML): Дата, Кол-во в партии.

Доступ к базе — живьём по имени вкладки (без выгрузки руками):
`/gviz/tq?tqx=out:csv&sheet=BASE`.

## Установка

```bash
pip3 install -r requirements.txt          # pillow, segno

cp config.example.json config.json        # и впиши ID своей таблицы
# ID берётся из URL таблицы между /d/ и /edit
# либо без файла:  export RASPIL_SHEET_ID=<id>
```
`config.json` в репозиторий НЕ попадает (в .gitignore) — ссылка на таблицу не публикуется.

## Генератор

**Полная динамика из таблицы** (рекомендуется) — скрипт сам читает задание
из вкладки РАСПИЛ и делает все бирки; ничего вводить руками не нужно:
```bash
python3 generate_labels.py --job 08.12.2025      # все детали за дату
python3 generate_labels.py --job all             # всё задание целиком
python3 generate_labels.py --offline --job 08.12.2025   # из кэша, без интернета
```
Откуда что: **РАСПИЛ** → какие детали, сколько штук, дата (из колонок Д1/Д2/Д3
формата `Имя-Nшт | ячейка`). **BASE** → материал, **Место**, размеры, CNC.

**Ручной режим** (отдельные детали):
```bash
python3 generate_labels.py "ANT-Bok L/R;27"
python3 generate_labels.py --date 08.12.2025 "ANT-Bok L/R;27" "V-Bok L/R;36"
```
Формат элемента: `"Имя"` или `"Имя;кол-во"`. Дата — через `--date dd.mm.yyyy`.

Логика «динамики»: **поменял «Место» в таблице → перегенерировал → на бирке новая ячейка.**
Проверено: правка `E3 → A5` в базе меняет надпись на бирке на `A5`.

### Зависимости
`pip3 install pillow segno` (для проверки QR — опционально `opencv-python-headless`).
Шрифты — системные Arial (есть в macOS). Иконка CNC — `assets/cnc_prisadka.png`
(вырезана 1:1 из оригинальной бирки).

### Точность формата
Выходной `.emf` побайтово того же формата, что у станка: BMP `1419×884×1`,
`cbSize 159182`, две копии подряд (итог 318 364 Б). QR читается и Apple Vision, и OpenCV,
payload как в оригинале (`ANT-Bok L-R` + хвостовой U+2800 для плотности/надёжности).

## Что ещё нужно от Ильнура, чтобы автоматизировать ПОЛНОСТЬЮ

Сейчас генератор делает бирки по списку названий. Чтобы он сам собирал папку `labels/`
точно под конкретное задание реза, нужно:

1. **Один реальный XML** задания + соответствующая папка `labels/` целиком — чтобы понять,
   как именно станок именует/привязывает бирки (label1, label2… по порядку реза? по ID?).
2. **Файл шаблона** (тот «Xbeer», где задаётся кодировка бирки) — подтвердить полный список
   полей и какие из них ещё динамические, кроме ячейки.
3. Подтвердить: **«№ячейки» = колонка «Место»** в BASE (в образце было «48» — это старое/тестовое?).
4. Список материалов для коротких подписей (`ЛДСП Белый Молет → ЛДСП «Белый»` и т.д.) —
   мэппинг в `MATERIAL_DISPLAY` в начале скрипта.
5. Разные ли иконки CNC под разные операции (присадка / паз / …) — пока одна.
